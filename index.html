**Zero dependências externas** (exceto Cascadia Code via CDN fontsource) - **Canvas API** para geração de imagens - **@font-face CSS** para Montserrat local - **GitHub Pages** para deploy --- ## ✅ AVATAR — Estado final (TRAVADO, não mexer) ### Funcionalidades - Upload drag & drop + clique - **Halftone sempre ativo** (toggle removido) — slider de pontos 3–12px - Toggle **Circular** (default checked) / Quadrado - Toggle **Logos** (default checked) - Campo **Handle / nome** com stroke 40% em Montserrat ### Modo Circular - Clip de círculo aplicado a tudo: foto + handle + logos - **Logos em arco** (`drawLogosArcSegments`) — 4 segmentos independentes com ícones canvas desenhados no ângulo do arco - **Handle em arco** (`arcText` com `skipBg=true`) — sem strip de fundo, só stroke+fill - Ordem de draw: foto → logos → handle (handle fica por cima) - Handle radius: `sz/2 * 0.70` com logos, `sz/2 * 0.91` sem logos - Logo arc radius: `sz/2 * 0.87` ### Modo Quadrado - Logos barra reta (`drawLogos` com `circ=false`) - Handle com stroke 40% Montserrat, sem barra de fundo ### Download - 2 variantes: **Fundo Limão** (bg=#EAFD7C, fg=#C299EE) e **Fundo Roxo** (bg=#C299EE, fg=#EAFD7C) - PNG 800×800 --- ## ✅ HERO DO SITE — Estado final (TRAVADO, não mexer) - dy²)` | | `szW is not defined` no avatar | `szW` escapou do escopo do `renderBanner` IIFE para dentro do `renderAvatar` | Revertido para `sz` simples no avatar | | Banner trava ao gerar | `document.getElementById('fntSans')` retornava `null` (elemento removido) e `.checked` jogava TypeError | Removida a referência ao `fntSans` inexistente | | Montserrat não carregava (font promise hanging) | `FontFace.load()` travava sem resolver nem rejeitar | Removido todo async font loading; usa apenas `@font-face` CSS com arquivo local | | Logos sumindo no landscape | `drawLogos` recebia `szH` na posição errada dos parâmetros (como `fg`) | Corrigida assinatura: `drawLogos(ctx, sz, fg, bg, circ, ht)` | | drawWaveHT com elipses | `var w` interno sombreava o parâmetro `w` (JS hoisting) | Renomeado para `var wv` interno | | Banner não renderizava | IIFE dentro de `renderBanner` — `szW` declarado dentro mas referenciado fora | Isolado corretamente dentro do IIFE | | Fonte do canvas com `;` | `ctx.font = '...' + fontFace + ';'` — canvas não aceita `;` | Removido o `;` da string de fonte | | Stroke invisível no handle circular | `strokeStyle = bg` (mesma cor do arc strip) | `skipBg=true` no handle: remove strip, usa só stroke+fill | --- ## 📐 Funções principais (JS) ```javascript // Avatar drawPhotoHT(img, ctx, sz, bg, fg, inv, circ, gs) // halftone da foto drawHandle(ctx, sz, txt, fg, bg, hasLg, circ) // handle/nome drawLogos(ctx, sz, fg, bg, circ, ht) // logos (circ=arc, false=barra) drawLogosArcSegments(ctx, sz, fg, bg) // logos em arco c/ ícones arcText(ctx, text, cx, cy, r, fg, bg, fs, arcCenter, arcSpan, fontFam, strokeW, skipBg) // Banner drawBannerTxt(ctx, sz, szH, t1, t2, t3, fg, bg, hasLg, pos) drawBannerPhoto(img, ctx, sz, szH, fg, inv, gs, pos) drawWaveHT(ctx, w, h, bg, fg, gs) drawUniformDots(ctx, w, h, bg, fg, gs) drawGradientDots(ctx, w, h, bg, fg, gs) drawStroked(text, x, y, strokeW) // stroke+fill helper dentro de drawBannerTxt ``` --- ## ⏳ O que ainda falta ajustar -
